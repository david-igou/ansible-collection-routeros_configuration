# `poe` role Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an imperative `poe` role for ad-hoc PoE-out operations — `power_cycle`, `power_off`, `power_on`, and read-only `monitor` — as the operational counterpart to the stateful `configure` role.

**Architecture:** A standard API role mirroring `reboot`/`ping`: `delegate_to: localhost` + `connection: local`, connection supplied via a `module_defaults` block over the shared `routeros_api_*` contract. Single-action-per-include interface (`routeros_poe_action` + `routeros_poe_interfaces`) so the play's host loop is the fan-out, fitting the per-host "PoE-port-as-hostvar" inventory model. `power_off`/`power_on` use `community.routeros.api_find_and_modify` (find-by-name, `require_matches_min: 1`); `monitor`/`power_cycle` resolve interface→`.id` from a path read, then run the keyed-menu command via `community.routeros.api` `cmd`. Persistent PoE knobs (watchdog, priority, voltage, LLDP, budget) stay in `configure`.

**Tech Stack:** Ansible role, `community.routeros.api` / `community.routeros.api_find_and_modify`, molecule (QEMU CHR, runs locally), ansible-lint, antsibull-docs (docsite via `meta/argument_specs.yml`).

**Spec:** `docs/superpowers/specs/2026-06-06-poe-role-design.md`

**Testing reality:** CHR (the molecule device) has no PoE controller, so live PoE ops cannot be exercised in CI. The molecule `poe` scenario covers the hardware-independent paths only (input validation, the no-entry failure, read-only monitor-all → empty). Live ops are validated later on real hardware. The scenario is written to pass whether CHR exposes an empty `/interface ethernet poe` menu OR omits it entirely (block/rescue on the menu-touching paths).

### Task 3: `tasks/main.yml` (the role logic)

**Files:**
- Create: `roles/poe/tasks/main.yml`

- [ ] **Step 1: Write `roles/poe/tasks/main.yml`**

```yaml
---
# tasks file for david_igou.routeros_configuration.poe
# Imperative PoE-out operations over the binary API. Not idempotent on the action
# paths: power_cycle is a one-shot command; power_off/power_on report changed
# honestly via api_find_and_modify; monitor is read-only. Persistent PoE config
# (watchdog, priority, voltage, lldp, global budget) belongs in `configure`.
#
# monitor/power_cycle are keyed-menu COMMANDS, so they need the entry .id (the
# api `cmd` takes key=value only — no CLI positionals / [find]); a path read maps
# interface name -> .id. power_off/power_on set the poe-out FIELD, so
# api_find_and_modify (find-by-name, require_matches_min:1) is cleanest and fails
# when the interface has no PoE-out entry. Results land under `.msg`.
- name: PoE-out operations
  delegate_to: localhost
  connection: local
  module_defaults:
    community.routeros.api:  # noqa: args[module]
      hostname: "{{ routeros_api_hostname }}"
      username: "{{ routeros_api_username }}"
      password: "{{ routeros_api_password }}"
      tls: "{{ routeros_api_tls | bool }}"
      validate_certs: "{{ routeros_api_validate_certs | bool }}"
      port: "{{ routeros_api_port | default(omit, true) }}"
    community.routeros.api_find_and_modify:  # noqa: args[module]
      hostname: "{{ routeros_api_hostname }}"
      username: "{{ routeros_api_username }}"
      password: "{{ routeros_api_password }}"
      tls: "{{ routeros_api_tls | bool }}"
      validate_certs: "{{ routeros_api_validate_certs | bool }}"
      port: "{{ routeros_api_port | default(omit, true) }}"
  block:
    # --- Validation (fires before any device read or mutation) ---
    - name: Require target interfaces for write actions
      ansible.builtin.assert:
        that:
          - routeros_poe_interfaces | length > 0
        fail_msg: >-
          routeros_poe_action={{ routeros_poe_action }} requires a non-empty
          routeros_poe_interfaces list
        quiet: true
      when: routeros_poe_action in ['power_cycle', 'power_off', 'power_on']

    - name: Validate the power_cycle duration
      ansible.builtin.assert:
        that:
          - routeros_poe_duration | string is match('^[0-9]+(ms|s|m)?$')
        fail_msg: >-
          routeros_poe_duration ({{ routeros_poe_duration }}) must be a RouterOS
          duration such as 5s, 30s, or 1m (0..1m)
        quiet: true
      when: routeros_poe_action == 'power_cycle'

    # --- power_off / power_on: set the poe-out field via find-and-modify ---
    - name: Set poe-out state
      community.routeros.api_find_and_modify:  # noqa: args[module]
        path: interface ethernet poe
        find:
          name: "{{ item }}"
        values:
          poe-out: "{{ 'off' if routeros_poe_action == 'power_off' else routeros_poe_mode }}"
        require_matches_min: 1
      loop: "{{ routeros_poe_interfaces }}"
      loop_control:
        label: "{{ item }}"
      when: routeros_poe_action in ['power_off', 'power_on']

    # --- monitor / power_cycle: resolve .id, then run the keyed-menu command ---
    - name: Read PoE-out entries (resolve interface -> .id)
      community.routeros.api:  # noqa: args[module]
        path: interface ethernet poe
      register: _poe_entries
      changed_when: false
      when: routeros_poe_action in ['monitor', 'power_cycle']

    - name: Fail if a named interface has no PoE-out entry
      ansible.builtin.assert:
        that:
          - (_poe_entries.msg | selectattr('name', 'equalto', item) | list | length) > 0
        fail_msg: "no PoE-out entry for {{ item }}"
        quiet: true
      loop: "{{ routeros_poe_interfaces }}"
      loop_control:
        label: "{{ item }}"
      when:
        - routeros_poe_action in ['monitor', 'power_cycle']
        - routeros_poe_interfaces | length > 0

    - name: Select target PoE-out entries
      ansible.builtin.set_fact:
        _poe_targets: >-
          {{ _poe_entries.msg
             if (routeros_poe_action == 'monitor' and routeros_poe_interfaces | length == 0)
             else (_poe_entries.msg | selectattr('name', 'in', routeros_poe_interfaces) | list) }}
      when: routeros_poe_action in ['monitor', 'power_cycle']

    - name: Power-cycle PoE-out ports
      community.routeros.api:  # noqa: args[module]
        path: interface ethernet poe
        cmd: >-
          power-cycle .id={{ item['.id'] }}
          duration={{ routeros_poe_duration | community.routeros.quote_argument_value }}
      loop: "{{ _poe_targets }}"
      loop_control:
        label: "{{ item['name'] }}"
      changed_when: true
      when: routeros_poe_action == 'power_cycle'

    - name: Monitor PoE-out ports
      community.routeros.api:  # noqa: args[module]
        path: interface ethernet poe
        cmd: "monitor .id={{ item['.id'] }} once"
      loop: "{{ _poe_targets }}"
      loop_control:
        label: "{{ item['name'] }}"
      register: _poe_monitor
      changed_when: false
      when: routeros_poe_action == 'monitor'

    - name: Expose PoE-out monitor results (routeros_poe_monitor)
      ansible.builtin.set_fact:
        routeros_poe_monitor: "{{ _poe_monitor.results | map(attribute='msg') | list }}"
      when: routeros_poe_action == 'monitor'

    - name: Show PoE-out monitor results
      ansible.builtin.debug:
        var: routeros_poe_monitor
      when: routeros_poe_action == 'monitor'
```

```markdown
# poe

Imperative PoE-out operations on a RouterOS device over the binary API:
power-cycle a port, force `poe-out` off/on, or read live `monitor` status.

This is the operational counterpart to the `configure` role — **not** a
replacement. Persistent PoE configuration (the ping watchdog
`power-cycle-ping-*`, periodic `power-cycle-interval`, `poe-priority`,
`poe-voltage`, `poe-lldp-enabled`, and the global `/interface ethernet poe
settings` power budget) is desired-state and belongs in `configure` via
`api_modify`. This role performs the one-shot actions that have no state to
reconcile.

## Actions (`routeros_poe_action`)

| Action | Effect |
| --- | --- |
| `monitor` (default) | Read-only live status (`poe-out-status`, voltage, current, power). Empty `routeros_poe_interfaces` reads all PoE-out ports. Results in the `routeros_poe_monitor` fact. |
| `power_cycle` | Drop power to the target ports for `routeros_poe_duration` (0..1m), then restore — power-cycles the attached devices. |
| `power_off` | Set `poe-out=off` on the target ports (cut power; reversible via `power_on`). |
| `power_on` | Restore `poe-out` to `routeros_poe_mode` (`auto-on` by default). |

Write actions (`power_cycle`/`power_off`/`power_on`) require a non-empty
`routeros_poe_interfaces`; a named interface with no PoE-out entry fails clearly.

## Variables

See `meta/argument_specs.yml` for the full list (and three worked examples in the
rendered role reference). Key variables: `routeros_poe_action`,
`routeros_poe_interfaces`, `routeros_poe_duration`, `routeros_poe_mode`, plus the
shared `routeros_api_*` connection variables.

## Per-host (PoE-port-as-hostvar) model

Target the powered hosts and point the connection at each host's switch — the
host loop is the fan-out:

\```yaml
- hosts: poe_powered_devices          # each has poe_switch + poe_port hostvars
  gather_facts: false
  tasks:
    - name: Power-cycle this host's port on its switch
      ansible.builtin.include_role:
        name: david_igou.routeros_configuration.poe
      vars:
        routeros_api_hostname: "{{ poe_switch }}"
        routeros_api_password: "{{ hostvars[poe_switch].routeros_api_password }}"
        routeros_poe_action: power_cycle
        routeros_poe_interfaces:
          - "{{ poe_port }}"
\```

## Note on hardware / testing

PoE-out is hardware-specific. The CHR used in CI has no PoE controller, so the
live actions are validated on real hardware; CI covers the input-validation and
no-entry paths.
```

```yaml
---
- name: Converge — poe validation + read-only monitor (CHR has no PoE)
  hosts: molecule
  gather_facts: false
  tasks:
    # Each block sets its flag false at the END (reached only if the role did NOT
    # fail) and true in rescue (reached when the role fails). The final assert
    # requires every flag true, so a role that WRONGLY accepts bad input fails.

    # 1. write action with no interfaces -> empty-interface guard
    - name: power_off with no interfaces is rejected
      block:
        - name: Attempt power_off with empty interfaces
          ansible.builtin.include_role:
            name: david_igou.routeros_configuration.poe
          vars:
            routeros_poe_action: power_off
            routeros_poe_interfaces: []
        - name: poe wrongly accepted empty interfaces
          ansible.builtin.set_fact:
            _poe_empty_rejected: false
      rescue:
        - name: Expected — empty-interface guard fired
          ansible.builtin.set_fact:
            _poe_empty_rejected: true

    # 2. invalid action -> argument-spec choices
    - name: invalid action is rejected
      block:
        - name: Attempt a bogus action
          ansible.builtin.include_role:
            name: david_igou.routeros_configuration.poe
          vars:
            routeros_poe_action: bogus
        - name: poe wrongly accepted a bogus action
          ansible.builtin.set_fact:
            _poe_action_rejected: false
      rescue:
        - name: Expected — action choices validation failed
          ansible.builtin.set_fact:
            _poe_action_rejected: true

    # 3. invalid mode -> argument-spec choices
    - name: invalid mode is rejected
      block:
        - name: Attempt power_on with a bogus mode
          ansible.builtin.include_role:
            name: david_igou.routeros_configuration.poe
          vars:
            routeros_poe_action: power_on
            routeros_poe_interfaces:
              - ether1
            routeros_poe_mode: bogus
        - name: poe wrongly accepted a bogus mode
          ansible.builtin.set_fact:
            _poe_mode_rejected: false
      rescue:
        - name: Expected — mode choices validation failed
          ansible.builtin.set_fact:
            _poe_mode_rejected: true

            routeros_poe_duration: 99x

          ansible.builtin.set_fact:
            _poe_duration_rejected: false

          ansible.builtin.include_role:
            name: david_igou.routeros_configuration.poe
          vars:
            routeros_poe_action: power_off
            routeros_poe_interfaces:
              - ether1
        - name: poe wrongly accepted a non-PoE interface
          ansible.builtin.set_fact:
            _poe_noentry_rejected: false

        - name: Expected — no PoE-out entry failure
          ansible.builtin.set_fact:
            _poe_noentry_rejected: true

    # 6. read-only monitor of all ports -> empty on a CHR (or menu absent)
    - name: monitor all PoE-out ports (read-only)
